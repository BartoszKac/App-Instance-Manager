BackRunner is designed so you can go from a blank browser tab to live program output in just a few steps. This page walks you through the happy path: opening the app, creating a project, writing code, compiling it, launching it, and reading the real-time output in the terminal panel. No compiler, runtime, or IDE needs to be installed on your machine.

<Note>
  BackRunner must be running before you follow these steps. If you have not set it up yet, see the [Self-Hosting guide](/self-hosting) to start the server locally in under two minutes with Docker Compose.
</Note>

## Your First Program

<Steps>

  <Step title="Open BackRunner in your browser">
    Navigate to the BackRunner URL in any modern browser. If you are running it locally with the default configuration, that is:

    ```text
    http://localhost:8888
    ```

    You will land on the **Projects** view, which lists all saved projects. On a fresh install the list is empty.
  </Step>

  <Step title="Create a new project">
    Click the **New Project** button. A dialog will prompt you to choose a language and give the project a name.

    Select your language — **Java**, **Python**, **C++**, or **Bash** — and enter a name. BackRunner creates the project and opens it immediately in the **Editor** view with a blank first file. The file is named after your project with the correct extension for the language you chose (for example `Main.java` for a Java project).

    <Tip>
      The first file in a project is always treated as the entry point when compiling and running. If you add more files later, the backend compiles them all together but starts execution from this initial file.
    </Tip>
  </Step>

  <Step title="Write your code">
    Type or paste code into the editor. The editor provides syntax highlighting for all four supported languages. Below are working examples for each language — pick the one that matches the language you selected.

    **Java**
    ```java title="Main.java"
    public class Main {
        public static void main(String[] args) {
            System.out.println("Hello from BackRunner!");
            for (int i = 1; i <= 5; i++) {
                System.out.println("Count: " + i);
            }
        }
    }
    ```

    **Python**
    ```python title="main.py"
    print("Hello from BackRunner!")
    for i in range(1, 6):
        print(f"Count: {i}")
    ```

    **C++**
    ```cpp title="main.cpp"
    #include <iostream>

    int main() {
        std::cout << "Hello from BackRunner!" << std::endl;
        for (int i = 1; i <= 5; i++) {
            std::cout << "Count: " << i << std::endl;
        }
        return 0;
    }
    ```

    **Bash**
    ```bash title="main.sh"
    #!/usr/bin/env bash
    echo "Hello from BackRunner!"
    for i in 1 2 3 4 5; do
        echo "Count: $i"
    done
    ```
  </Step>

  <Step title="Compile your code">
    Click the **Compile** button in the editor toolbar.

    For Java, BackRunner sends all files in the project to the server, which runs `javac --release 17 -d . *.java`. For C++, it invokes `g++ -Wall` across all `.cpp` files. Python and Bash skip the compilation step entirely — BackRunner recognises them as interpreted languages and marks the project ready to run immediately.

    If compilation succeeds, the toolbar shows a success indicator and the **Run** button becomes active. If there are errors, the backend returns the compiler's full error output, which appears in the terminal panel at the bottom of the screen so you can see exactly what went wrong.

    <Warning>
      Java source files must declare a class with the same name as the file. For example, code in `Main.java` must contain `public class Main { ... }`. This is a Java language requirement, not a BackRunner restriction.
    </Warning>
  </Step>

  <Step title="Run your program and watch the output">
    Click the **Run** button. BackRunner launches your compiled program on the server and opens a WebSocket connection to the `/topic/output` channel. Every line your program writes to stdout or stderr appears in the terminal panel in real time, with no page refresh required.

    For the examples above you should see:

    ```
    Hello from BackRunner!
    Count: 1
    Count: 2
    Count: 3
    Count: 4
    Count: 5
    ```

    The terminal panel remains open after the program exits, so you can scroll through the full output at any time.

    <Tip>
      The terminal panel is always visible at the bottom of the screen across all views. It continues to display the latest output even if you navigate to the Deploy or Instances view.
    </Tip>
  </Step>

</Steps>

## What to Explore Next

Now that you have run your first program, here is where to go depending on what you want to do.

<CardGroup cols={2}>
  <Card title="Add More Files" icon="file-plus">
    Use the **Add File** button in the editor to create additional source files in the same project. The backend compiles all files together on the next compile action, letting you split code across multiple classes or translation units.
  </Card>
  <Card title="Deploy to a Remote Server" icon="server">
    Navigate to the **Deploy** view to register a remote Linux server and push your compiled artifact over SSH. You need the server's hostname, port, username, and password.
  </Card>
  <Card title="Upload a Pre-Built Binary" icon="upload">
    If you have a compiled executable that was built outside BackRunner, use the **Upload** view to register it. Uploaded binaries appear in the Instances view and can be deployed to remote servers just like compiled projects.
  </Card>
  <Card title="Manage Running Instances" icon="list-check">
    The **Instances** view shows every remote program configuration. Select one, provide the shell command to launch it, and BackRunner executes it on the target server over SSH.
  </Card>
</CardGroup>
