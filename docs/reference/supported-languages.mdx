BackRunner supports four programming language types. Java and C++ go through a compilation step before execution; Python is interpreted and runs directly without any build phase. Bash is recognised as a language type and can be edited in the IDE, but its execution handler is not yet implemented — running a `.sh` file will return an error. The `UNKNOWN` type is reserved for files whose extension cannot be matched to a supported language — you cannot execute files in that state.

## Language overview

| Language | Enum value | Extension | Default filename | Compiled? |
|----------|-----------|-----------|-----------------|-----------|
| Java     | `JAVA`    | `.java`   | `Main.java`     | ✅ Yes (`javac`) |
| Python   | `PYTHON`  | `.py`     | `main.py`       | ❌ No (interpreted) |
| C++      | `CPP`     | `.cpp`    | `main.cpp`      | ✅ Yes (`g++`) |
| Bash     | `BASH`    | `.sh`     | `main.sh`       | ❌ No (interpreted) — *execution not yet available* |

## Per-language reference

<AccordionGroup>
  <Accordion title="Java" icon="mug-hot">
    BackRunner compiles Java projects with `javac` targeting **Java 17**. All `.java` files in the project are passed to the compiler in a single invocation — the **first file you created** is treated as the entry point, and its class name is used for the `java` run command.

    **Compile command**
    ```bash
    javac --release 17 -d . Main.java OtherClass.java ...
    ```

    **Run command**
    ```bash
    java -cp . Main
    ```

    <Note>
      Multi-file support is unique to Java. You can add as many `.java` source files as you need to a project and they will all be compiled together. Make sure your entry-point class contains a `public static void main(String[] args)` method.
    </Note>

    **Hello, World!**
    ```java
    public class Main {
        public static void main(String[] args) {
            System.out.println("Hello, World!");
        }
    }
    ```
  </Accordion>

  <Accordion title="Python" icon="snake">
    Python projects skip the compilation step entirely. BackRunner invokes `python3` on Linux/macOS servers and falls back to `python` on Windows hosts. Any additional `.py` files you add to the project are available for import as long as they live in the same project directory.

    **Run command**
    ```bash
    python3 main.py
    ```

    <Tip>
      You can use standard `import` statements to reference other `.py` files in your project. Because they share the same working directory, no special path configuration is needed.
    </Tip>

    **Hello, World!**
    ```python
    print("Hello, World!")
    ```
  </Accordion>

  <Accordion title="C++" icon="c">
    BackRunner compiles C++ sources with `g++`. If the environment variable `GPP_PATH` is set on the server, that path is used instead of the default `g++` binary — useful when the server has multiple compiler versions installed. The `-Wall` flag is always included to surface common warnings in the output panel.

    **Compile command**
    ```bash
    g++ main.cpp -o main -Wall
    ```

    **Run command**
    ```bash
    ./main
    ```

    <Note>
      On Windows servers the output binary is given an `.exe` extension automatically and run without the `./` prefix. You do not need to adjust your code for this — BackRunner handles it internally.
    </Note>

    **Hello, World!**
    ```cpp
    #include <iostream>

    int main() {
        std::cout << "Hello, World!" << std::endl;
        return 0;
    }
    ```
  </Accordion>

  <Accordion title="Bash" icon="terminal">
    Bash scripts are run directly with the `bash` interpreter — no compilation step is involved. This language is best suited for automation scripts, shell utilities, and quick command-line tasks you want to prototype and test in the browser.

    **Run command**
    ```bash
    bash main.sh
    ```

    <Warning>
      Bash execution support is defined at the language-type level but the execution handler is not yet implemented in the current release. Attempting to run a `.sh` file will return an unsupported-language error. You can still create and edit Bash files — execution support is planned for a future release.
    </Warning>

    **Hello, World!**
    ```bash
    #!/usr/bin/env bash
    echo "Hello, World!"
    ```
  </Accordion>
</AccordionGroup>

## The `UNKNOWN` type

If BackRunner cannot match a file's extension to one of the four supported languages, the file is assigned the `UNKNOWN` type internally. Files in this state **cannot be compiled or executed**. To resolve this, rename your file to use a supported extension (`.java`, `.py`, `.cpp`, or `.sh`) and re-open it in the editor.

<Warning>
  Attempting to run a project whose primary file has the `UNKNOWN` type will result in an error from the backend. Always verify the file extension before clicking **Run**.
</Warning>
