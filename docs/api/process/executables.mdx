Remote executables are pre-compiled binaries or raw scripts that you upload directly to BackRunner. Unlike source-code workflows, there is no compilation step — you send the file as-is, tag it with a language type, and BackRunner stores it ready to be dispatched to a runner instance. Every executable is persisted with its raw byte contents, its original filename, and the `LanguageType` that tells the runner how to invoke it.

<Note>
  The maximum file size for a single upload is **150 MB**. The same 150 MB ceiling also applies to the total HTTP request size, so keep metadata and form fields small.
</Note>

---

## The RemoteExecutable object

All endpoints that return executable data use the following JSON shape.

```json title="RemoteExecutable"
{
  "id": 42,
  "name": "solution.py",
  "language": "PYTHON",
  "contents": "<base64-encoded bytes>"
}
```

<ResponseField name="id" type="integer">
  Auto-generated unique identifier assigned by the server on creation. Read-only.
</ResponseField>

<ResponseField name="contents" type="string (base64)">
  Raw bytes of the uploaded file, base64-encoded in JSON responses. When you submit an executable via the `/bulk` endpoint, send the bytes as a base64 string.
</ResponseField>

<ResponseField name="language" type="LanguageType (string enum)" required>
  The language runtime that will be used to execute the file. Must be one of:

  | Value | Description |
  |---|---|
  | `JAVA` | Compiled Java bytecode (`.class` / `.jar`) |
  | `PYTHON` | Python script (`.py`) |
  | `CPP` | C++ binary (`.cpp` or pre-compiled) |
  | `BASH` | Shell script (`.sh`) |
  | `UNKNOWN` | Any other file type — runner behaviour is best-effort |
</ResponseField>

<ResponseField name="name" type="string" required>
  The original filename, including the extension. Set automatically from the uploaded file's name when you use `POST /upload`.
</ResponseField>

## Upload an executable

<Warning>
  The upload endpoint consumes `multipart/form-data`. Sending `application/json` will result in a `415 Unsupported Media Type` error. Make sure your HTTP client sets the correct `Content-Type` header, or lets the multipart boundary be set automatically (as `curl` does with `-F`).
</Warning>

Uploads a single file and stores it as a new `RemoteExecutable`. The file's original name is preserved automatically. On success the server returns `201 Created` with a plain-text confirmation message.

```
POST /api/process/upload
```

### Query parameters

<ParamField query="language" type="LanguageType" required>
  The language type for the uploaded file. Must be one of `JAVA`, `PYTHON`, `CPP`, `BASH`, or `UNKNOWN`.
</ParamField>

### Request body

Send the file as a `multipart/form-data` part named **`file`**.

### Example — upload a Python script

```bash
curl -X POST "https://your-backrunner-host/api/process/upload?language=PYTHON" \
  -F "file=@/path/to/solution.py"
```

### Example — upload a compiled Java JAR

```bash
curl -X POST "https://your-backrunner-host/api/process/upload?language=JAVA" \
  -F "file=@/path/to/app.jar"
```

### Example — upload a Bash script

```bash
curl -X POST "https://your-backrunner-host/api/process/upload?language=BASH" \
  -F "file=@/path/to/run.sh"
```

### Response

```
HTTP/1.1 201 Created
Content-Type: text/plain
```

```
Executable 'solution.py' saved successfully.
```

<Tip>
  If you do not know the language ahead of time, pass `language=UNKNOWN`. You can update the language field later with `PUT /api/process/{id}`.
</Tip>

## Bulk-create executables

Creates multiple `RemoteExecutable` records in a single request. Useful when you want to seed a runner environment with a set of pre-built files at once. Returns `200 OK` with a plain-text summary.

```
POST /api/process/bulk
```

### Request body

Send a JSON array of `RemoteExecutable` objects. The `id` field is ignored — the server assigns new IDs. `contents` must be the raw file bytes encoded as a base64 string.

```json
[
  {
    "name": "solution.py",
    "language": "PYTHON",
    "contents": "cHJpbnQoJ2hlbGxvIHdvcmxkJyk="
  },
  {
    "name": "runner.sh",
    "language": "BASH",
    "contents": "IyEvYmluL2Jhc2gKZWNobyAiSGVsbG8i"
  }
]
```

### Example

```bash
curl -X POST "https://your-backrunner-host/api/process/bulk" \
  -H "Content-Type: application/json" \
  -d '[
    {
      "name": "solution.py",
      "language": "PYTHON",
      "contents": "cHJpbnQoJ2hlbGxvIHdvcmxkJyk="
    },
    {
      "name": "runner.sh",
      "language": "BASH",
      "contents": "IyEvYmluL2Jhc2gKZWNobyAiSGVsbG8i"
    }
  ]'
```

### Response

```
HTTP/1.1 200 OK
Content-Type: text/plain
```

```
2 executable(s) saved successfully.
```

## List all executables

Returns the full list of every `RemoteExecutable` stored in the system.

```
GET /api/process
```

### Example

```bash
curl -X GET "https://your-backrunner-host/api/process" \
  -H "Accept: application/json"
```

### Response

```json
[
  {
    "id": 1,
    "name": "solution.py",
    "language": "PYTHON",
    "contents": "cHJpbnQoJ2hlbGxvIHdvcmxkJyk="
  },
  {
    "id": 2,
    "name": "runner.sh",
    "language": "BASH",
    "contents": "IyEvYmluL2Jhc2gKZWNobyAiSGVsbG8i"
  }
]
```

## Get an executable by ID

Retrieves the `RemoteExecutable` matching the given `id`. The response is always a **list** (even when only one record matches), keeping the return shape consistent with the list-all endpoint.

```
GET /api/process/{id}
```

### Path parameters

<ParamField path="id" type="integer" required>
  The unique identifier of the executable you want to retrieve.
</ParamField>

### Example

```bash
curl -X GET "https://your-backrunner-host/api/process/1" \
  -H "Accept: application/json"
```

### Response

```json
[
  {
    "id": 1,
    "name": "solution.py",
    "language": "PYTHON",
    "contents": "cHJpbnQoJ2hlbGxvIHdvcmxkJyk="
  }
]
```

<Note>
  Even when fetching by ID, the response is wrapped in a JSON array. Always read index `[0]` to access the matched executable.
</Note>

## Update an executable

Replaces the stored fields of an existing `RemoteExecutable`. You can update the name, language, and/or contents in a single call. Returns `200 OK` with a plain-text confirmation.

```
PUT /api/process/{id}
```

### Path parameters

<ParamField path="id" type="integer" required>
  The unique identifier of the executable to update.
</ParamField>

### Request body

A full `RemoteExecutable` JSON object. All fields you want to retain must be included — omitted fields may be overwritten with null values.

<ParamField body="name" type="string" required>
  The filename, including extension.
</ParamField>

<ParamField body="language" type="LanguageType" required>
  Updated language type. One of `JAVA`, `PYTHON`, `CPP`, `BASH`, `UNKNOWN`.
</ParamField>

<ParamField body="contents" type="string (base64)" required>
  Raw file bytes, base64-encoded.
</ParamField>

### Example — change the language of an existing record

```bash
curl -X PUT "https://your-backrunner-host/api/process/1" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "solution.py",
    "language": "PYTHON",
    "contents": "cHJpbnQoJ2hlbGxvIHdvcmxkJyk="
  }'
```

### Response

```
HTTP/1.1 200 OK
Content-Type: text/plain
```

```
Executable with id 1 updated successfully.
```

## Delete an executable by ID

Permanently removes the `RemoteExecutable` with the given `id`. Returns `204 No Content` with an empty body on success.

```
DELETE /api/process/{id}
```

### Path parameters

<ParamField path="id" type="integer" required>
  The unique identifier of the executable to delete.
</ParamField>

### Example

```bash
curl -X DELETE "https://your-backrunner-host/api/process/1"
```

### Response

```
HTTP/1.1 204 No Content
```

---

## Delete all executables

Removes every `RemoteExecutable` from the system in one operation. Returns `200 OK` with a plain-text confirmation message.

<Warning>
  This operation is irreversible. All stored executables and their file contents will be permanently deleted.
</Warning>

```
DELETE /api/process
```

### Example

```bash
curl -X DELETE "https://your-backrunner-host/api/process"
```

### Response

```
HTTP/1.1 200 OK
Content-Type: text/plain
```

```
All executables deleted successfully.
```
