The Code API gives you full control over the lifecycle of a BackRunner project, from storing and editing source files all the way through compilation and execution. Every operation is scoped to a **project**, which is identified by a single integer called `idManClass` — the `id` of that project's main-class (entry-point) source file. All other source files in the project carry the same `idManClass` so BackRunner knows which files belong together.

<Note>
  The Code API requires **no authentication**. All endpoints accept cross-origin requests. Do not expose your BackRunner instance on a public network without adding an authentication layer at the infrastructure level.
</Note>

## Base URL

```
http://localhost:8888
```

Replace `localhost` with the hostname or IP address of your BackRunner server.

## The `idManClass` identifier

`idManClass` is the `id` of the main-class `SourceCode` record — the entry point of your project (for example, `Main.java` or `main.py`). It acts as a **project namespace**: every source file, compiled artefact, and launch operation uses it to group related records. You receive the value of `idManClass` when you first save a project's source files via `POST /api/code/source` — the `id` that the database assigns to the main-class file becomes the project's `idManClass`.

## Endpoint summary

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/code/source/main-class/{idMainClass}` | Get all source files for a project |
| `GET` | `/api/code/source/all` | Get every source file across all projects |
| `GET` | `/api/code/source/main-class/all` | Get only the main-class (entry-point) files |
| `POST` | `/api/code/source` | Save a list of source files; returns the disk path |
| `PUT` | `/api/code/source` | Update a single source file |
| `DELETE` | `/api/code/project/{idMainClass}` | Delete an entire project and all its files |
| `GET` | `/api/code/compiled/main-class/{idMainClass}` | Get compiled artefacts for a project |
| `POST` | `/api/code/compile/all/main-class/{idMainClass}` | Compile all source files in a project |
| `POST` | `/api/code/launch/main-class/{idMainClass}` | Launch a compiled project |

## Data models

### SourceCode

A `SourceCode` object represents a single source file stored in BackRunner.

```json
{
  "id": 1,
  "name": "Main.java",
  "code": "public class Main { public static void main(String[] args) { } }",
  "language": "JAVA",
  "idManClass": 1
}
```

| Field | Type | Description |
|-------|------|-------------|
| `id` | `Long` | Auto-generated primary key. Use `0` when creating a new file. |
| `name` | `String` | File name including extension, e.g. `Main.java`. |
| `code` | `String` | Full source code content. |
| `language` | `LanguageType` | One of `JAVA`, `PYTHON`, `CPP`, `BASH`, `UNKNOWN`. |
| `idManClass` | `Long` | Project identifier — the `id` of the main-class file. |

### CompiledCode

A `CompiledCode` object represents a compiled output file stored by BackRunner after a build.

```json
{
  "idCode": 10,
  "name": "Main.class",
  "code": "<binary-or-bytecode-representation>",
  "language": "JAVA",
  "idManClass": 1
}
```

| Field | Type | Description |
|-------|------|-------------|
| `idCode` | `Long` | Auto-generated primary key for the compiled record. |
| `name` | `String` | Output file name, e.g. `Main.class`. |
| `code` | `String` | Compiled output content stored by BackRunner. |
| `language` | `LanguageType` | Language of the source file that produced this artefact. |
| `idManClass` | `Long` | Project identifier — matches the `idManClass` you queried. |

### LanguageType values

| Value | File extension |
|-------|---------------|
| `JAVA` | `.java` |
| `PYTHON` | `.py` |
| `CPP` | `.cpp` |
| `BASH` | `.sh` |
| `UNKNOWN` | _(none)_ |
