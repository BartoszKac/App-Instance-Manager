Once your project has compiled successfully, you launch it with a single `POST` request. BackRunner spawns the program as a managed process on the server and immediately returns a status string. From that point on, all stdout and stderr output from the running process is streamed to connected WebSocket clients in real time.

<Note>
  Confirm that BackRunner has stored compiled artefacts for your project by checking `GET /api/code/compiled/main-class/{idMainClass}` before calling this endpoint. See [Compile](/api/code/compile) for details.
</Note>

---

## POST /api/code/launch/main-class/{idMainClass}

Starts the compiled project identified by `idMainClass`. BackRunner selects the correct runtime (JVM, Python interpreter, C++ binary, or shell) based on the project's `LanguageType`.

### Path parameters

<ParamField path="idMainClass" type="Long" required>
  The project identifier. Must match a project whose source files have been saved and compiled.
</ParamField>

### Response

Returns a plain `String` — either a process identifier or a short status message. The response body is not JSON.

```bash
curl -X POST http://localhost:8888/api/code/launch/main-class/1
```

```
process-1719834205123
```

<Tip>
  Store the returned process identifier if you need to correlate WebSocket output to a specific run, or to display run status in your UI.
</Tip>

## Streaming output via WebSocket

After the process starts, BackRunner publishes all stdout and stderr to a STOMP WebSocket topic. Connect to the WebSocket endpoint and subscribe to the output topic to receive log lines as they are emitted.

**WebSocket endpoint:** `ws://localhost:8888/ws`

**Subscription topic:** `/topic/output`

Each message on `/topic/output` is a plain text string containing one or more lines of program output.

```js
import { Client } from "@stomp/stompjs";

const client = new Client({
  brokerURL: "ws://localhost:8888/ws",
  onConnect: () => {
    client.subscribe("/topic/output", (message) => {
      console.log("Output:", message.body);
    });
  },
});

client.activate();
```

<Warning>
  Replace `localhost` with your BackRunner host in both the REST and WebSocket URLs. If you are running BackRunner behind a reverse proxy, ensure the proxy is configured to forward WebSocket upgrade requests.
</Warning>

For the full WebSocket event reference including connection lifecycle and error topics, see the [WebSocket reference](/reference/websocket).
