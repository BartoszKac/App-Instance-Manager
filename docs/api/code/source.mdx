Source files are the raw code you write in the BackRunner editor. Each file belongs to exactly one project, identified by `idManClass`. The endpoints on this page let you retrieve files at any granularity — a single project, all projects, or only entry-point files — and give you full create, update, and delete control.

---

## GET /api/code/source/main-class/{idMainClass}

Returns all source files that belong to the given project.

### Path parameters

<ParamField path="idMainClass" type="Long" required>
  The project identifier. This is the `id` of the main-class `SourceCode` record.
</ParamField>

### Response

Returns a `List<SourceCode>`. Each object contains the fields described in the [SourceCode model](/api/code/overview#sourcecode).

```bash
curl http://localhost:8888/api/code/source/main-class/1
```

```json
[
  {
    "id": 1,
    "name": "Main.java",
    "code": "public class Main {\n  public static void main(String[] args) {\n    System.out.println(\"Hello, BackRunner!\");\n  }\n}",
    "language": "JAVA",
    "idManClass": 1
  },
  {
    "id": 2,
    "name": "Helper.java",
    "code": "public class Helper {\n  public static String greet() { return \"Hi\"; }\n}",
    "language": "JAVA",
    "idManClass": 1
  }
]
```

## GET /api/code/source/all

Returns every source file stored in BackRunner, across all projects.

```bash
curl http://localhost:8888/api/code/source/all
```

```json
[
  {
    "id": 1,
    "name": "Main.java",
    "code": "...",
    "language": "JAVA",
    "idManClass": 1
  },
  {
    "id": 3,
    "name": "main.py",
    "code": "print('hello')",
    "language": "PYTHON",
    "idManClass": 2
  }
]
```

<Tip>
  Use `GET /api/code/source/main-class/all` instead if you only want the entry-point file for each project — this is useful for building project-list UIs without fetching every helper file.
</Tip>

## GET /api/code/source/main-class/all

Returns only the main-class (entry-point) source files — one per project. These are the files whose `id` equals the `idManClass` of their project.

## POST /api/code/source

Saves a list of source files to the BackRunner filesystem and database. Use this when creating a new project or persisting a multi-file workspace for the first time.

### Request body

A JSON array of `SourceCode` objects. All files in the array must share the same `idManClass`.

<ParamField body="id" type="Long">
  Leave as `0` (or omit) when creating new files. Provide the existing `id` to overwrite a record.
</ParamField>

<ParamField body="name" type="String" required>
  File name including extension, e.g. `Main.java`.
</ParamField>

<ParamField body="code" type="String" required>
  Full source code content for this file.
</ParamField>

<ParamField body="language" type="LanguageType" required>
  Programming language. One of: `JAVA`, `PYTHON`, `CPP`, `BASH`, `UNKNOWN`.
</ParamField>

<ParamField body="idManClass" type="Long" required>
  Project identifier. All objects in the array must use the same value.
</ParamField>

### Response

Returns a `String` — the absolute filesystem path where the project was written on the BackRunner host.

```bash
curl -X POST http://localhost:8888/api/code/source \
  -H "Content-Type: application/json" \
  -d '[
    {
      "id": 0,
      "name": "Main.java",
      "code": "public class Main {\n  public static void main(String[] args) {\n    System.out.println(\"Hello!\");\n  }\n}",
      "language": "JAVA",
      "idManClass": 1
    }
  ]'
```

```
/opt/backrunner/projects/1
```

<Note>
  The returned path is the root directory of the project on the BackRunner host. You do not need this path for subsequent API calls — use `idManClass` instead.
</Note>

## PUT /api/code/source

Updates a single existing source file. The file to update is identified by the `id` field in the request body.

### Request body

A single `SourceCode` object with all fields populated.

<ParamField body="id" type="Long" required>
  The `id` of the source file to update.
</ParamField>

<ParamField body="name" type="String" required>
  File name including extension.
</ParamField>

<ParamField body="code" type="String" required>
  Updated source code content.
</ParamField>

<ParamField body="language" type="LanguageType" required>
  One of: `JAVA`, `PYTHON`, `CPP`, `BASH`, `UNKNOWN`.
</ParamField>

<ParamField body="idManClass" type="Long" required>
  Project identifier. Must match the original value stored for this file.
</ParamField>

### Response

Returns the updated `SourceCode` object.

<ResponseField name="id" type="Long">
  The primary key of the updated source file.
</ResponseField>

<ResponseField name="name" type="String">
  File name including extension.
</ResponseField>

<ResponseField name="code" type="String">
  The updated source code content.
</ResponseField>

<ResponseField name="language" type="LanguageType">
  Language of the file. One of `JAVA`, `PYTHON`, `CPP`, `BASH`, `UNKNOWN`.
</ResponseField>

<ResponseField name="idManClass" type="Long">
  Project identifier the file belongs to.
</ResponseField>

```bash
curl -X PUT http://localhost:8888/api/code/source \
  -H "Content-Type: application/json" \
  -d '{
    "id": 1,
    "name": "Main.java",
    "code": "public class Main {\n  public static void main(String[] args) {\n    System.out.println(\"Updated!\");\n  }\n}",
    "language": "JAVA",
    "idManClass": 1
  }'
```

```json
{
  "id": 1,
  "name": "Main.java",
  "code": "public class Main {\n  public static void main(String[] args) {\n    System.out.println(\"Updated!\");\n  }\n}",
  "language": "JAVA",
  "idManClass": 1
}
```

## DELETE /api/code/project/{idMainClass}

Permanently deletes an entire project — all source files, compiled artefacts, and the on-disk project directory — identified by `idMainClass`.

<Warning>
  This action is irreversible. All source code and compiled outputs for the project are deleted immediately. Make sure you have exported or committed your code elsewhere before calling this endpoint.
</Warning>

### Path parameters

<ParamField path="idMainClass" type="Long" required>
  The project identifier to delete.
</ParamField>

### Response

Returns `204 No Content` on success with an empty body.

```bash
curl -X DELETE http://localhost:8888/api/code/project/1
```

```
HTTP/1.1 204 No Content
```
