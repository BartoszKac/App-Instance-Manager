Compilation in BackRunner happens entirely on the server — your browser sends a request and the backend invokes the appropriate toolchain against the source files you have saved. You never need to install a compiler locally. The process is on-demand: nothing compiles until you explicitly click the **Compile** button, giving you full control over when your code is built.

## Which languages require compilation?

BackRunner treats its four supported languages in two groups:

<CardGroup cols={2}>
  <Card title="Compiled languages" icon="gear">
    **Java** and **C++** must be compiled before they can run. The **⚙ Compile** button appears in the editor toolbar only for these projects. The **Run** button stays hidden until compilation succeeds.
  </Card>
  <Card title="Interpreted languages" icon="play">
    **Python** and **Bash** do not require a separate compile step. The **Run** button becomes available as soon as you save your files for the first time. BackRunner still validates these files server-side when they are executed.
  </Card>
</CardGroup>

## How to compile

<Steps>
  <Step title="Save your files">
    Click **Save** in the top-right toolbar to push the latest source to the server. The compiler always works against the files stored on the backend, not against what is currently in the editor buffer.
  </Step>
  <Step title="Click ⚙ Compile">
    The button label changes to **Compiling…** and becomes disabled while the build runs. BackRunner sends a `POST` request to:
    ```
    POST /api/code/compile/all/main-class/{idMainClass}
    ```
    This instructs the server to compile every source file associated with the project in one pass.
  </Step>
  <Step title="Read the result">
    A short status message appears next to the toolbar buttons:
    - A **green** message confirms the build succeeded.
    - A **red** message indicates one or more files failed to compile.
  </Step>
  <Step title="Run your program (on success)">
    Once the status is green the **▶ Run** button appears. Click it to launch your compiled program and stream its output to the terminal panel.
  </Step>
</Steps>

## Compiler details by language

<Tabs>
  <Tab title="Java">
    BackRunner uses **`javac`** to compile Java source files. All `.java` files in the project are compiled together in a single invocation so that classes can reference each other freely.

    ```java
    // Entry-point file — must contain the class with main()
    public class Main {
        public static void main(String[] args) {
            Greeter g = new Greeter();
            g.greet();
        }
    }
    ```

    ```java
    // Additional file: Greeter.java
    public class Greeter {
        public void greet() {
            System.out.println("Hello from Greeter!");
        }
    }
    ```

    <Note>
      The entry-point class — the one containing `public static void main(String[] args)` — must live in the first file you created when you set up the project.
    </Note>
  </Tab>
  <Tab title="C++">
    BackRunner uses **`g++`** to compile C++ source files. All `.cpp` files in the project are passed to the compiler together, so you can split your code across multiple translation units.

    ```cpp
    // Entry-point file — must contain main()
    #include <iostream>
    #include "utils.h"

    int main() {
        printMessage("Hello from BackRunner!");
        return 0;
    }
    ```

    ```cpp
    // Additional file: utils.cpp
    #include <iostream>
    #include "utils.h"

    void printMessage(const std::string& msg) {
        std::cout << msg << std::endl;
    }
    ```
  </Tab>
  <Tab title="Python">
    Python does **not** require a compile step in BackRunner. After you save your files the **▶ Run** button appears immediately. The Python interpreter validates your syntax at runtime; any syntax errors are reported as red lines in the terminal output panel.

    ```python
    # main.py — entry point
    from utils import greet

    if __name__ == "__main__":
        greet("BackRunner")
    ```
  </Tab>
  <Tab title="Bash">
    Bash scripts do **not** require a compile step. Save your files and click **▶ Run**. The shell interpreter reports errors inline in the terminal output.

    ```bash
    #!/usr/bin/env bash
    # main.sh — entry point
    source ./helpers.sh
    say_hello "BackRunner"
    ```
  </Tab>
</Tabs>

## Compile status reference

After compilation completes, BackRunner fetches the per-file results from `GET /api/code/compiled/main-class/{idMainClass}` and checks whether any file has a `status` of `"error"`.

| Status indicator | Colour | Meaning |
|-----------------|--------|---------|
| `success` | Green (`#4ade80`) | All files compiled without errors; Run is now available |
| `error`   | Red (`#f87171`)   | At least one file has a compile error |
| Compiling | *(button disabled)* | Build is in progress |
| Idle      | *(no message)* | No compile has been triggered yet this session |

## Reading compile errors

When compilation fails, check the terminal output panel at the bottom of the editor. The server streams the raw compiler output — `javac` and `g++` both emit the filename, line number, and a description of the problem in a standard format:

```
Main.java:5: error: ';' expected
        System.out.println("Hello")
                                   ^
1 error
```

Error lines appear in **red** in the terminal panel. Fix the reported line in the editor, click **Save**, and then click **⚙ Compile** again to re-run the build.

<Tip>
  If you see the message "Compile error" in the toolbar but the terminal panel is empty, make sure you have clicked **Connect** in the terminal panel to establish the WebSocket connection before running your next compile.
</Tip>
