BackRunner's editor is built on the Monaco engine — the same engine that powers Visual Studio Code. When you open a project you get the full Monaco experience inside your browser: syntax highlighting, bracket matching, IntelliSense-style autocomplete, and multi-cursor editing, all without installing anything locally. The editor occupies the central area of the screen, flanked by a file sidebar on the left and a status bar along the bottom edge.

## Editor layout

<CardGroup cols={3}>
  <Card title="Sidebar" icon="sidebar">

  <Card title="Tab bar" icon="table-columns">

## The entry-point file

Every BackRunner project has one **entry-point file** — the first file created when you set up the project. The compiler and runtime target this file as the program's starting point, so you should keep your `main` method (Java/C++) or top-level script logic (Python/Bash) there.

<Info>
  The entry-point file is always the first file created for a project. You cannot change which file is the entry point after creation; structure your code so that the initial file delegates to the rest of your project.
</Info>

## Adding files to a project

You can add as many source files to a project as you need.

<Steps>
  <Step title="Click the + icon in the sidebar header">
    A small inline input field appears at the bottom of the sidebar file list.
  </Step>
  <Step title="Type the filename including its extension">
    Use the correct extension for your language (`.java`, `.py`, `.cpp`, or `.sh`). For example: `Utils.java` or `helpers.py`. BackRunner infers the language from the extension automatically.
  </Step>
  <Step title="Press Enter or click Add">
    The file is added to the project and immediately opened in a new tab. Its content starts empty.
  </Step>
</Steps>

<Tip>
  Press `Escape` while the filename input is focused to cancel adding a file without creating anything.
</Tip>

To remove a file, hover over its name in the sidebar and click the × icon that appears on the right. Removing a file closes its tab and deletes it from the in-memory project state. The deletion becomes permanent the next time you save.

## Saving your work

Click the **Save** button in the top-right toolbar to persist all project files to the server. BackRunner sends a `POST /api/code/source` request containing every file in the project. While saving is in progress the button label changes to **Saving…** and becomes disabled.

If anything goes wrong, a red error message appears next to the toolbar buttons. After a successful first save the **Run** button becomes available for Python and Bash projects (compiled languages need a successful compile step first).

<Warning>
  BackRunner does not auto-save. Remember to click **Save** before you compile or run your code, otherwise the server will execute an older version of your files.
</Warning>

To clear the contents of the currently active file without deleting the file itself, click the **Clear** button in the toolbar.

## Code examples by language

Below are minimal starter examples for each supported language. You can paste these directly into the entry-point file of a new project.

<Tabs>
  <Tab title="Java">
    ```java
    public class Main {
        public static void main(String[] args) {
            System.out.println("Hello from BackRunner!");
        }
    }
    ```
  </Tab>
  <Tab title="Python">
    ```python
    def main():
        print("Hello from BackRunner!")

    if __name__ == "__main__":
        main()
    ```
  </Tab>
  <Tab title="C++">
    ```cpp
    #include <iostream>

    int main() {
        std::cout << "Hello from BackRunner!" << std::endl;
        return 0;
    }
    ```
  </Tab>
  <Tab title="Bash">
    ```bash
    #!/usr/bin/env bash
    echo "Hello from BackRunner!"
    ```
  </Tab>
</Tabs>

## VS Code-style editor features

Because Monaco is the underlying engine, you get the same keyboard shortcuts and editing conveniences you already know from VS Code:

| Feature | Detail |
|---------|--------|
| Syntax highlighting | Colour-coded for Java, Python, C++, and Bash out of the box |
| Autocomplete | IntelliSense suggestions appear as you type; accept with `Tab` or `Enter` |
| Bracket matching | Matching brackets are highlighted when the cursor is adjacent to one |
| Multi-cursor | Hold `Alt` and click to place additional cursors; `Ctrl+D` selects the next occurrence |
| Find & Replace | `Ctrl+H` opens the inline find-and-replace panel |
| Line numbers | Displayed in the gutter; click a number to select the whole line |
