BackRunner gives you a full development environment in your browser. Write code with the Monaco Editor (the same engine that powers VS Code), compile and run it on a remote server on demand, and watch stdout and stderr stream back to you live. When your code is ready, deploy it to any SSH-accessible machine in a few clicks.

## What you can do with BackRunner

<CardGroup cols={3}>
  <Card title="Write & Compile" icon="pen-to-square">
    Edit multi-file projects in the browser and compile them server-side without redeploying the application.
  </Card>
  <Card title="Stream Output" icon="terminal">
    See live stdout and stderr from your running program via a built-in WebSocket terminal panel.
  </Card>
  <Card title="Deploy via SSH" icon="upload">
    Transfer source code, compiled bytecode, or pre-built binaries directly to any SSH server you configure.
  </Card>
  <Card title="Manage Instances" icon="circle-play">
    View all deployed instances in one place and run ad-hoc shell commands on each remote server.
  </Card>
  <Card title="Multi-language" icon="layer-group">
    Java, Python, C++, and Bash are supported out of the box. Each language gets its own compilation pipeline.
  </Card>
  <Card title="REST API" icon="plug">
    Every feature is exposed through a documented REST API so you can automate or integrate BackRunner into CI workflows.
  </Card>
</CardGroup>

## Get started

<Steps>
  <Step title="Set up BackRunner">
    Deploy the BackRunner server using Docker Compose or run it locally with Maven. See the [Self-Hosting guide](/self-hosting) for full instructions.
  </Step>
  <Step title="Create a project">
    Open the Projects screen, click **New Project**, choose a language, and BackRunner scaffolds the entry-point file for you.
  </Step>
  <Step title="Write and compile your code">
    Use the Monaco Editor to write your code. Hit **Compile** to build it on the server. Errors appear inline in the output panel.
  </Step>
  <Step title="Run and watch output live">
    Click **Run** and watch stdout stream directly to the terminal panel in real time over WebSocket.
  </Step>
</Steps>

<Tip>
  You can also upload a pre-compiled binary or executable directly through the **Upload** screen — no code editing required.
</Tip>
