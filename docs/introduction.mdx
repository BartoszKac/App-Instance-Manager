BackRunner is a full-featured, browser-based IDE that lets you write, compile, and run code on a live server without installing compilers, runtimes, or development tooling on your local machine. You open the app, write your code in the built-in editor, hit **Run**, and watch output stream back to your screen in real time — all from any browser tab. This page introduces what BackRunner does, who it is designed for, and how its three main views work together.

## What BackRunner Does

BackRunner gives you a self-contained code execution environment powered by a Spring Boot backend and a React frontend. When you submit code, the server compiles it (where the language requires it), spawns the resulting process, and pipes every line of stdout and stderr back to your browser over a persistent WebSocket connection. You never wait for a page reload to see results — output appears as soon as your program produces it.

Beyond writing and running code, BackRunner includes a **Deploy** view that pushes compiled artifacts to remote Linux servers over SSH, and an **Instances** view that lets you launch and monitor those remote programs. A separate **Upload** workflow accepts pre-built executables up to 150 MB so you can deploy binaries that were compiled outside of BackRunner.

## Supported Languages

BackRunner ships with four built-in language handlers. Each language is supported end-to-end: source files are accepted, compiled where required, and executed on the server.

| Language | File Extension | Compilation Step |
|----------|---------------|------------------|
| Java     | `.java`       | `javac --release 17` |
| Python   | `.py`         | None (interpreted) |
| C++      | `.cpp`        | `g++ -Wall` |
| Bash     | `.sh`         | None (interpreted) |

## Core Capabilities

  <Card title="Code Editor" icon="code">
    A lightweight in-browser code editor with syntax highlighting for all four supported languages, powered by Prism.js. Write and edit your source files directly in the browser — no local IDE or toolchain needed.
  </Card>
  <Card title="Real-Time Output" icon="bolt">
    Program stdout and stderr are streamed live to the browser via WebSocket (STOMP protocol) on the `/topic/output` topic. Long-running programs update the terminal incrementally — you see every line the moment it is printed.
  </Card>
  <Card title="Multi-File Projects" icon="folder-open">
    Group related source files into a single project. The backend tracks all files together and compiles them as a unit, treating the first file you create as the entry point. Useful for Java classes spread across multiple files or C++ programs with separate translation units.
  </Card>
  <Card title="SSH Deployment" icon="server">
    Push compiled artifacts to any SSH-accessible Linux server directly from the Deploy view. BackRunner stores remote server credentials and program configurations so you can re-deploy with a single action.
  </Card>
  <Card title="Instance Management" icon="list-check">
    The Instances view shows all programs currently registered on remote servers. Select a configuration, provide a run command, and launch or inspect your program without leaving the browser.
  </Card>
  <Card title="File Upload" icon="upload">
    Already have a compiled binary? Use the Upload view to push pre-built executables (up to 150 MB per file) into the BackRunner registry, then deploy them via the same SSH workflow.
  </Card>
</CardGroup>

## Technology Stack

BackRunner is built on two independently runnable projects that ship together.

**Backend** — A Spring Boot 3.2 application written in Java 17, exposing a REST API under `/api` and a WebSocket endpoint at `/ws-console`. It uses an embedded H2 file-based database, so no external database server is required. SSH file transfers are handled by JSch. The OpenAPI UI is available at `/swagger-ui.html` when the server is running.

**Frontend** — A React 19 + TypeScript application bundled with Vite. The code editor is provided by `react-simple-code-editor` backed by Prism.js for syntax highlighting, and the WebSocket client uses SockJS and StompJS to connect to the backend.
